## Task 1: Fix hop 2 riskyIncomingVolume formula

**Files:**
- Modify: `backend/src/modules/address-check/address-check.service.ts:1079`
- Test: `backend/src/modules/address-check/__tests__/taint-model.test.ts`

The current code at line 1079 does `riskyIncomingVolume += pathShare * totalVolume`.
`pathShare = alpha * beta` where `alpha = h1.incomingVolume / totalVolume`. Expanding:
`pathShare * totalVolume = (h1.incomingVolume / totalVolume) * (tVol / vols2.totalVolume) * totalVolume = h1.incomingVolume * (tVol / vols2.totalVolume)`
This is wrong. The correct amount is just `tVol` — the actual volume from that hop-2 counterparty.

  // Simulate: totalVolume=1000, h1.incomingVolume=400, tVol=200, vols2.totalVolume=500
  // pathShare = (400/1000) * (200/500) = 0.16
  // WRONG old formula: pathShare * totalVolume = 0.16 * 1000 = 160
  // CORRECT: tVol = 200
  const totalVolume = 1000;
  const h1IncomingVolume = 400;
  const tVol = 200;
  const vols2TotalVolume = 500;

  assert.equal(wrongFormula, 160, 'old formula gives 160');
  assert.equal(correctFormula, 200, 'correct formula gives tVol directly');

## Task 4: Fix expansion race condition and add transaction wrapping

**Files:**
- Modify: `backend/src/modules/ingestion/expansion.service.ts:154-242`

Two bugs:
1. `crawlerExpansionAddresses` is a shared array that concurrent `ops` push to — race condition
2. `Promise.all(ops)` runs upserts without `prisma.$transaction` — partial commits on failure

The fix: move `crawlerExpansionAddresses.push(address)` out of the async `ops` callbacks and into a synchronous collection after upserts succeed, OR collect addresses from `entries` upfront (they're known before async work). Since `entries` is already computed synchronously before the `ops.map`, we can collect crawler addresses from `entries` directly without touching the async callbacks.

## Task 5: Fix graph crawler — prevent address re-enqueueing cycles

**Files:**
- Modify: `backend/src/modules/ingestion/graph-crawler.service.ts`

The crawler at line 719-727 calls `this.enqueueAddress(e.address)` without checking if `e.address` was already seen in this crawl run. The `CrawlQueue` table has `CrawlQueueStatus`, so addresses already `PROCESSING` or `DONE` won't cause infinite work, but they'll still cause redundant DB upserts and wasted queue writes in the same batch.

The real protection needed: within a single `processQueueItem` call, don't re-enqueue addresses that are the same as the current item's address (already done at line 1107 in the service: `if (uAddr === address || visitedAddresses.has(uAddr)) continue`).

For the crawler, add a simple check: don't enqueue if `e.address` equals `item.address` (the root being processed).

- [ ] **Step 4: Fix CORS and add helmet in app.ts**

Replace `backend/src/app.ts` with:

```typescript
import express, { Express } from 'express';
import helmet from 'helmet';
import healthRoutes from './modules/health/health.routes';
import checkRoutes from './modules/check/check.routes';
import { errorHandler, notFoundHandler } from './middleware/errorHandler';
import { apiRateLimiter } from './middleware/rateLimiter';
import { env } from './config/env';

const createApp = (): Express => {
  const app = express();

  // Security headers
  app.use(helmet());

  // CORS middleware
  app.use((req, res, next) => {
    const origin = req.headers.origin as string | undefined;
    const allowed =
      env.allowedOrigins.length === 0 ||
      (origin != null && env.allowedOrigins.includes(origin));

    if (allowed && origin) {
      res.header('Access-Control-Allow-Origin', origin);
    }
    res.header(
      'Access-Control-Allow-Methods',
      'GET, POST, OPTIONS'
    );
    res.header(
      'Access-Control-Allow-Headers',
      'Origin, X-Requested-With, Content-Type, Accept, x-api-key'
    );
    if (req.method === 'OPTIONS') {
      res.sendStatus(200);
    } else {
      next();
    }
  });

  // Middleware
  app.use(express.json({ limit: '1mb' }));
  app.use(express.urlencoded({ extended: true, limit: '1mb' }));

  // Global rate limiting
  app.use('/api', apiRateLimiter);

  // Routes
  app.use('/', healthRoutes);
  app.use('/api/v1/check', checkRoutes);

  // Error handling
  app.use(notFoundHandler);
  app.use(errorHandler);

  return app;
};

export default createApp;
```

Note: `env.allowedOrigins.length === 0` means "dev mode, allow all origins" — acceptable since auth bypass is now fixed separately. In prod, set `ALLOWED_ORIGINS=https://yourdomain.com`.
